## Problem Statement

Voice deepfakes are AI-generated audio recordings that closely mimic the
speech patterns, tone, and identity of a real person. Modern Text-to-Speech
(TTS) systems — such as ElevenLabs, Tortoise TTS, VITS, WaveNet, and
MelGAN — have reached a level of realism where they are nearly
indistinguishable to the human ear.

This project builds a **binary classifier** that answers one question:

```
Given an audio clip → is this a REAL human voice or an AI-GENERATED fake?
```

---

## Why This Matters

Voice deepfakes are actively being misused in:

- **Financial fraud** — attackers clone a CEO's voice to authorise wire
  transfers (multiple documented cases of $25M+ losses)
- **Political misinformation** — fabricated audio of politicians saying
  things they never said
- **Non-consensual audio cloning** — celebrities and private individuals
  having their voices cloned without consent
- **Authentication bypass** — breaking voice-based identity verification
  systems used by banks and call centres
- **Disinformation campaigns** — spreading false news using fake audio
  evidence

Detecting these fakes in real-time, before they cause harm, is an
active and critical area of AI safety research.

## Dataset

### Primary — Fake-or-Real (FoR)
- **Source:** Zenodo (https://zenodo.org/record/4275027)
- **Size:** ~111,000 audio clips
- **Content:** Real human speech recordings + speech generated by
  multiple TTS systems (Google TTS, Amazon Polly, Microsoft Azure,
  IBM Watson, and others)
- **Format:** 16kHz mono WAV files
- **Licence:** Open for research use

### Secondary — WaveFake
- **Source:** GitHub (RUB-SysSec/WaveFake)
- **Size:** ~117,000 audio clips
- **Content:** Speech processed through 6 neural vocoders:
  MelGAN, HiFi-GAN, WaveGlow, WaveNet, Parallel WaveGAN, MB-MelGAN
- **Why useful:** Each vocoder leaves different GAN artifacts in
  the spectrogram — the model learns to detect all of them

### Synthetic Fallback
If real datasets cannot be downloaded, `download_dataset.py`
programmatically generates synthetic audio using `numpy` and `scipy`.

- **Real samples:** Natural harmonic series with pitch wobble,
  irregular amplitude envelope, and pink background noise
- **Fake samples:** Perfect harmonic series (no natural variation),
  frame-boundary click artifacts, unnaturally flat amplitude,
  sharp high-frequency cutoff

These synthetic samples produce meaningfully different spectrograms,
allowing the model to train even without internet access.

### Dataset Split

| Split       | Proportion | Purpose                              |
|-------------|-----------|--------------------------------------|
| Training    | 70%       | Model learns weights from these      |
| Validation  | 15%       | Tuning + early stopping decisions    |
| Test        | 15%       | Final honest evaluation (never seen) |

Stratified splitting ensures each split has the same real/fake ratio.

## Audio Preprocessing Pipeline

Every audio file — regardless of original format, sample rate, or
length — is converted to a standardised numpy array before any
ML processing occurs.

```
load_and_preprocess(filepath)
         │
         ├─ librosa.load(sr=16000, mono=True)
         │    Resample to 16kHz, collapse stereo → mono
         │
         ├─ librosa.effects.trim(top_db=20)
         │    Remove leading/trailing silence
         │    (top_db=20: cut anything 20dB below peak)
         │
         ├─ Amplitude normalisation
         │    audio = audio / max(|audio|) × 0.9
         │    Ensures consistent signal level across files
         │
         └─ Fixed-length clipping / padding
              If  len > 48000: take from centre
              If  len < 48000: zero-pad at end
              Result: always exactly 48000 samples (3 seconds at 16kHz)
```

**Why 16kHz?**
Human speech occupies 80Hz–8000Hz. By Nyquist's theorem, a 16kHz
sample rate captures all frequencies up to 8kHz — the complete
speech range. Using higher rates wastes memory without adding
useful information for speech analysis.

**Why 3 seconds?**
Long enough to capture multiple syllables and the temporal patterns
(like GAN frame-boundary artifacts). Short enough to keep the
feature matrix a manageable, fixed size for batched CNN training.

## Feature Extraction

Two types of features are extracted from the preprocessed audio:

### 1. MFCC Feature Vector (used by baseline model)

**What are MFCCs?**
Mel-Frequency Cepstral Coefficients describe the short-term power
spectrum of a sound. They capture the shape of the vocal tract —
the resonant frequencies that give each voice its unique character.

**Extraction process:**
```
audio (48000 samples)
  → Short-Time Fourier Transform (STFT) with 2048-point windows
  → 40 mel filter banks applied to each frame
  → Log compression (mimics ear's logarithmic loudness perception)
  → DCT to decorrelate → 40 MFCC coefficients per frame
  → Compute mean of each coeff across time  → (40,) vector
  → Compute std  of each coeff across time  → (40,) vector
  → Concatenate mean + std                  → (80,) feature vector
```

**Why mean AND std?**
The mean captures "what kind of voice is this overall."
The std captures how much variation exists — GAN voices tend to
have unnaturally low standard deviation (too consistent = fake).

**Additional waveform features (4 values):**
- Zero Crossing Rate — how often the signal crosses zero
- RMS Energy — overall loudness
- Spectral Centroid — centre of gravity of frequency energy
- Spectral Bandwidth — spread of frequency energy

**Total baseline feature vector: 80 + 4 = 84 dimensions**

### 2. Mel Spectrogram (used by CNN)

A mel spectrogram is a 2D image where:
- X axis = time (left to right)
- Y axis = frequency on mel scale (bottom = low, top = high)
- Pixel brightness = energy at that frequency at that time

```
audio (48000 samples)
  → Mel spectrogram: 128 mel bands × ~93 time frames
  → Power-to-dB conversion (log scale)
  → Resize to 128 × 128 pixels (PIL bilinear interpolation)
  → Normalise to [0, 1] range
  → Output: (128, 128) float32 array
```

**GAN artifacts visible in mel spectrograms:**

| Artifact | Description | Cause |
|----------|-------------|-------|
| Periodic vertical stripes | Faint lines at regular intervals | GAN frame-boundary state reset |
| Sharp high-freq cutoff | Abrupt silence above ~4kHz | Generator models noise poorly |
| Horizontal stripe pattern | Ghost harmonics at fixed intervals | Conv filter periodicity |
| Excessive mid-freq smoothness | Too uniform 500Hz–2kHz band | Generator cannot model irregular formants |

Real speech, by contrast, shows organic, irregular, noisy patterns
throughout all frequency bands with no repeating structure.

## Model Architectures

### Model 1 — Baseline (RandomForest)

```
Input: 84-dimensional feature vector (MFCC + waveform stats)
  └─ StandardScaler (normalise to mean=0, std=1)
     └─ RandomForestClassifier
          n_estimators  = 200   (200 independent decision trees)
          max_depth     = 20    (each tree asks ≤ 20 questions)
          class_weight  = balanced
          n_jobs        = -1    (use all CPU cores)
Output: P(fake) ∈ [0, 1]
```

Training time: ~1–3 minutes on CPU
Advantage: Instant training, interpretable, good baseline

### Model 2 — Main Model (AudioCNN)

```
Input: (batch, 1, 128, 128) mel spectrogram tensor

Conv Block 1:  Conv2d(1→32, 3×3)  → BatchNorm2d → ReLU → MaxPool(2×2)
               Output: (batch, 32, 64, 64)

Conv Block 2:  Conv2d(32→64, 3×3) → BatchNorm2d → ReLU → MaxPool(2×2)
               Output: (batch, 64, 32, 32)

Conv Block 3:  Conv2d(64→128,3×3) → BatchNorm2d → ReLU → MaxPool(2×2)
               Output: (batch, 128, 16, 16)

Conv Block 4:  Conv2d(128→256,3×3)→ BatchNorm2d → ReLU → AvgPool(4×4)
               Output: (batch, 256, 4, 4)

Flatten:       (batch, 4096)

Dense 1:       Linear(4096→512) → ReLU → Dropout(0.5)
Dense 2:       Linear(512→128)  → ReLU → Dropout(0.3)
Output:        Linear(128→1)    → Sigmoid

Output: P(fake) ∈ (0, 1)
```

**Total trainable parameters:** ~2.1M

**Key design decisions:**

- `BatchNorm2d` after every Conv layer — stabilises training,
  allows higher learning rates, reduces sensitivity to weight init
- `Dropout(0.5)` — randomly zeroes 50% of neurons during training,
  forcing the network to learn redundant representations (prevents
  overfitting on small datasets)
- `AdaptiveAvgPool2d(4,4)` in Block 4 — forces fixed output size
  regardless of input, making the model robust to slight size variation
- `Sigmoid` output — squeezes the raw logit to a probability in (0,1),
  compatible with Binary Cross-Entropy loss

## Training Process

### Loss Function — Binary Cross-Entropy (BCE)

```
BCELoss(p, y) = -[y·log(p) + (1-y)·log(1-p)]

where:
  p = model's predicted probability (0–1)
  y = true label (0 = real, 1 = fake)

Perfect prediction: loss = 0
Worst prediction:   loss = very large positive number
```

### Optimiser — Adam

Adam (Adaptive Moment Estimation) adapts the learning rate individually
for each weight based on the history of gradients. It converges faster
and more reliably than plain SGD for this type of problem.

```
Learning rate:   0.001
Weight decay:    1e-4   (L2 regularisation)
```

### Learning Rate Scheduling

`ReduceLROnPlateau`: if validation loss does not improve for 4 consecutive
epochs, multiply the learning rate by 0.5. This helps the model escape
flat regions of the loss surface.

### Early Stopping

Stops training if validation loss does not improve for 8 consecutive
epochs. Saves the checkpoint from the best epoch.

**Why early stopping?**
Without it, the model trains until `MAX_EPOCHS` regardless of whether
it is still improving. After the optimal epoch, the model starts
memorising training data (overfitting) — validation accuracy drops
while training accuracy continues rising. Early stopping captures
the model at its best generalisation point.

### Data Augmentation (training only)

Applied randomly to each spectrogram during training:

| Technique | Probability | Effect |
|-----------|-------------|--------|
| Time masking | 40% | Blank out a random vertical band |
| Frequency masking | 40% | Blank out a random horizontal band |
| Gaussian noise | 30% | Add small random values to all pixels |
| Brightness jitter | 20% | Multiply all values by random 0.85–1.15 |

This makes each epoch slightly different, forcing generalisation.

## Results

### Baseline Model (RandomForest on MFCCs)

| Metric    | Value  |
|-----------|--------|
| Accuracy  | ~78%   |
| Precision | ~76%   |
| Recall    | ~81%   |
| F1-score  | ~78%   |
| ROC-AUC   | ~0.86  |

### CNN Model (spectrogram-based)

| Metric    | Value  |
|-----------|--------|
| Accuracy  | ~84%   |
| Precision | ~83%   |
| Recall    | ~86%   |
| F1-score  | ~84%   |
| ROC-AUC   | ~0.92  |

> Note: Exact results vary by dataset split and random seed.
> These figures are representative of training on ~800 samples.
> Larger datasets (full FoR or WaveFake) achieve 92–97% accuracy.

**CNN outperforms baseline because:**
The CNN reads the entire spectrogram image and learns spatial patterns
(like GAN stripe artifacts) that are invisible to the MFCC-based
feature vector. MFCCs collapse the time dimension — they cannot
capture the temporal periodicity of GAN frame boundaries.

## Installation

### Prerequisites

- Python 3.9 or above
- pip
- (Optional) ffmpeg — for non-WAV audio uploads

Verify Python:
```bash
python --version
# Should show: Python 3.9.x or above
```

Install ffmpeg (optional but recommended):
```bash
# macOS:
brew install ffmpeg

# Ubuntu/Debian:
sudo apt install ffmpeg

# Windows: download from https://ffmpeg.org/download.html
# and add the bin/ folder to your system PATH
```

```bash
# 1. Enter the project directory
cd audio-deepfake-detector

# 2. Create a virtual environment
python -m venv venv

# 3. Activate the virtual environment
# Windows (Command Prompt):
venv\Scripts\activate
# Windows (PowerShell):
venv\Scripts\Activate.ps1
# macOS / Linux:
source venv/bin/activate

# 4. Install all Python dependencies
pip install -r requirements.txt
```

## Future Improvements

1. **Larger datasets** — Training on the full FoR + WaveFake datasets
   (~200K samples) would push accuracy to 92–97%

2. **Transformer-based model** — Replace CNN with a vision transformer
   (ViT) or audio-specific model like wav2vec 2.0 for better long-range
   temporal dependency modelling

3. **Speaker-independent evaluation** — Test on speakers not seen
   during training to measure true generalisation

4. **Partial fake detection** — Detect spliced audio (real + fake
   segments mixed in one clip) — a harder and more realistic attack

5. **Real-time streaming** — Process live microphone input in 3-second
   sliding windows for real-time detection

6. **Explainability layer** — GradCAM attention heatmaps overlaid on
   the spectrogram to show exactly which frequency bands triggered the
   "fake" decision

7. **Adversarial robustness** — Test against adversarial attacks where
   the fake audio is post-processed to fool the detector

8. **Mobile deployment** — Export CNN to ONNX format for edge inference
   on mobile devices (< 50ms prediction on phone)
